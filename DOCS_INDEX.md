## ðŸŽ¯ Common Tasks

### How do I...

**...get started as a developer?**
1. Read [README - Quick Start](./README.md#quick-start)
2. Read [CONTRIBUTING - Getting Started](./CONTRIBUTING.md#getting-started)
3. Set up your environment using [.env.example](./.env.example)

**...understand how the system works?**
1. Read [ARCHITECTURE - Overview](./ARCHITECTURE.md#overview)
2. Read [ARCHITECTURE - Data Architecture](./ARCHITECTURE.md#data-architecture)
3. Read [docs/plugins.md](./docs/plugins.md) for platform integrations

**...fix a bug or issue?**
1. Check [DEBUGGING](./DEBUGGING.md) for common issues
2. Search [GitHub Issues](https://github.com/Krosebrook/fusion-stage-hub/issues)
3. Follow [CONTRIBUTING - Development Workflow](./CONTRIBUTING.md#development-workflow)

**...deploy to production?**
1. Read [DEPLOYMENT](./DEPLOYMENT.md) end-to-end
2. Set up environment variables from [.env.example](./.env.example)
3. Follow [DEPLOYMENT - Post-Deployment Checklist](./DEPLOYMENT.md#post-deployment-checklist)

**...report a security issue?**
1. Read [SECURITY - Reporting a Vulnerability](./SECURITY.md#reporting-a-vulnerability)
2. **DO NOT** create public GitHub issues
3. Email security team or use GitHub Security Advisory

**...add a new platform integration?**
1. Read [docs/plugins.md - Plugin Development](./docs/plugins.md#plugin-development)
2. Follow [CONTRIBUTING - Coding Standards](./CONTRIBUTING.md#coding-standards)
3. Submit PR following [CONTRIBUTING - Pull Request Process](./CONTRIBUTING.md#pull-request-process)

## ðŸ’¡ Documentation Best Practices

When writing documentation:

- **Be concise**: Get to the point quickly
- **Use examples**: Show, don't just tell
- **Add diagrams**: Text-based ASCII art works great
- **Link related docs**: Help readers navigate
- **Keep it current**: Update when code changes
- **Test instructions**: Verify steps actually work
